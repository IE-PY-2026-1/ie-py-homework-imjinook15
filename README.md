# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 지옥의 팀플 벌금 관리기
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 조별과제(팀플) 참여도에 따라 조원들의 지각비와 벌금을 공정하게 계산하고, 누적된 금액을 관리하는 '팀플 벌금 정산 시스템'입니다.
* 사용자는 조원의 이름과 지각 시간(분)을 입력하여 당일 발생한 기본 지각비를 산출합니다. 시스템은 이 지각 시간을 바탕으로 출석 상태(정상, 단순 지각, 무단결석 등)를 판별하고, 조건에 따라 차등적인 벌금을 부여합니다. 또한, '무임승차(결석)' 조건을 만족하는 불성실 조원에게는 가중 처벌(고액 벌금)을 내리는 로직이 포함됩니다. 최종적으로는 사용자가 메뉴판을 통해 지각자를 지속적으로 추가 기록하고, 배열을 활용해 여러 조원의 데이터를 동시에 관리하며 조원별 누적 벌금 현황과 전체 모인 회식비(총 벌금)를 한 번에 조회할 수 있도록 전체 시스템을 완성할 계획입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 조원 이름 및 지각 시간 입력
2. 당일 지각비 및 벌금 계산
3. 출석 상태 판별(우수/경고/지각/무임승차)
4. 조원별 누적 벌금 현황 출력
5. 현재까지 모인 전체 회식비(총 벌금) 조회
0. 프로그램 종료

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
(예시)
 1. **내용 1 : 지각비 관리 프로그램 컨셉 도출**
    * **프롬프트 요약:** ""파이썬으로 성적관리프로그램 같은 프로그램을 만들고 싶어. 일반적인 대학생에게 맞춰서 다른 독특하고 현실적인 아이디어 없을까?""
    * **적용 내용:** AI가 제안한 여러 아이디어 중, 대학생이라면 누구나 폭풍 공감할 수 있는 '조별과제 지각비 및 무임승차 관리 프로그램'이라는 컨셉이 가장 마음에 들어 채택함. 단순한 성적 관리를 넘어 '팀플 벌금을 공정하게 정산한다'는 명확한 목표를 세우니 기획이 훨씬 즐거워짐.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 조원 지각비 정산 데이터 구조 모델링 논의"
    * **적용 내용:** 과제 필수 요건을 맞추기 위해 포괄적인 변수를 사용하는 대신 조원의 데이터를 세분화함. 문자형(char - 조원 이니셜), 정수형(int - 학번 끝 4자리, 지각 시간), 실수형(float - 분당 벌금 기준액, 산출된 총 벌금액) 등 3가지 이상의 자료형을 활용하여 총 5개의 변수를 선언함으로써, 입력 데이터의 구조적 완성도를 높이고 1차 과제 평가 기준을 완벽히 충족하도록 설계함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 조별과제(팀플) 참여도에 따른 지각비 및 무임승차 정산 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 조건별 연산과 흐름 제어를 통해 최종 벌금 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 if-elif 조건문과 and 논리 연산자, break/continue를 활용하여 대학생의 현실적인 팀플 상황에 맞는 지능형 로직을 완성해야 합니다.
  * input()으로 조원 수와 각 조원의 이름을 차례대로 입력받아 리스트(names)에 저장.
  * if-elif-else를 사용하여 출석 상태(정상, 소폭 지각, 중증 지각, 무단결석)를 4단계로 세분화하여 판정하는 로직 추가.
  * and 연산자를 활용해 지각 시간(1~9분) 조건을 설정하고, 10분 이상 지각 시 실수형(float) 가중치(rate)를 곱하는 '벌금 폭탄' 기능 구현.
  * continue(0분 지각 시 계산 스킵)와 break(무단결석 시 정산 강제 종료)를 활용한 입체적인 프로그램 흐름 제어 적용.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "나 아직 while문 안 배웠어. 상태 입력에서 1이나 2를 안 적었을 때 while 없이 어떻게 처리하면 좋을까? 그냥 0원으로 넘어가기는 아쉬워."
    * **적용 내용:** 배운 문법만 사용해야 한다는 제약 조건 내에서, AI가 제안한 '진행 방해죄' 아이디어를 채택함. 오타나 장난으로 이상한 숫자를 입력할 경우 else문으로 빠지게 하여, 무한 루프(while)로 다시 묻는 대신 즉시 고액의 벌금(10,000원)을 확정 부과하는 방식으로 에러 대처를 시스템의 재미 요소로 승화시킴.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "흐름 제어에서 -99나 999 같은 인위적인 비상 코드 말고, 다른 방식으로 break와 continue를 자연스럽게 사용해주었으면 좋겠어. 예를 들어 결석 시 팀플이 파토나니까 break로 나가는 식으로 짜줘."
    * **적용 내용:** AI와 논의하여 억지스러운 임의의 숫자 입력을 없애고, '0분 지각' 시 계산을 건너뛰는 continue와 '무단결석' 발생 시 즉시 최대 벌금을 부과하고 프로그램 전체를 종료해버리는 break 로직을 도입함. 이를 통해 문법적 요구사항을 충족함과 동시에 '지옥의 팀플'이라는 기획 의도를 완벽하게 살림.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** 벌금 가중치를 실수형(float)으로 입력받아 연산하다 보니, 최종 벌금액이 '2500.0원'처럼 부자연스러운 소수점 형태로 출력됨.
    * **해결:** 연산이 끝난 최종 결과값 변수에 int() 함수를 씌워 fee = int(calc_fee) 형태로 강제 형변환(Type Casting)을 수행하여 깔끔한 원화 정수 단위로 출력되게 개선함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** 초기에는 for i in range(3):처럼 반복 횟수를 하드코딩하여 3명 고정으로만 조원을 입력받을 수 있었음.
    * **해결:** 조원의 수를 먼저 정수형 변수(num)로 input() 받은 뒤, 본인(기록자)을 먼저 리스트에 넣고 for i in range(num - 1):을 사용하여 남은 인원수만큼 유동적으로 반복 입력받도록 로직을 수정함.
